数据结构学习笔记

2023/5/3

绪论

数据结构的基本概念

数据结构三要素：

  1. 数据的逻辑结构；
  2. 数据的存储结构；
  3. 数据的运算。

可以用抽象数据类型定义一个完整的数据结构，抽象数据类型（ADT）描述了数据的逻辑结构和抽象运算，通常用（数据对象，数据关系，基本操作集）这样的三元组来表示，从而构成一个完整的数据结构定义。

有序表是指关键字有序的线性表，仅描述元素之间的逻辑关系，它既可以链式存储，又可以顺序存储，属于逻辑结构。

数据的逻辑结构独立于其存储结构，而数据的存储结构是逻辑结构在计算机上的映射，它不能独立于逻辑结构而存在。

链式存储设计时，各个不同结点的存储空间可以不连续，但结点内的存储单元地址必须连续。

算法和算法评价

线性表

// 定义单链表
typedef struct LNode
{
    int data;
    struct LNode *next;
} LNode, *LinkList;
// 初始化单链表（带头结点）
bool InitList(LinkList &L)
{
    L = (LNode *)malloc(sizeof(LNode));
    if (L == NULL)
    {
        return false;
    }
    L->next = NULL;
    return true;
}

LinkList 等价于 LNode * ，前者强调这是链表，后者强调这是结点。

初始化单链表，都要先把头指针指向NULL。

栈、队列和数组

栈

重要术语：

  • 栈顶：允许插入和删除的一端；
  • 栈底：不允许插入和删除的一端；
  • 空栈。

Last In First Out(LIFO)：后进先出。

逻辑结构上栈和普通线性表相同。

卡特兰数：
$$
\frac{1}{n+1}C{n \choose 2n} = 合法的出栈序列数
$$

栈和队列的应用

栈在括号匹配中的应用

​ 括号匹配的处理过程与栈的思想吻合。

栈在表达式求值中的应用

​ 中缀表达式：a + b

​ a + b - c * d

​ 后缀表达式（逆波兰式）： a b +

​ a b + c d * -

机算中，遵循左优先原则。

​ 前缀表达式（波兰式）： + a b

​ - + a b * c d

遵循右优先原则。

栈在递归中的应用

函数调用的特点：最后被调用的函数最先执行结束。

递回的精髓在于能否将原始问题转换为属性相同但规模较小的问题。

递归模型的条件：

  • 递归表达式
  • 边界条件

缺点：效率不高，递归次数过多容易造成栈溢出。

队列在层次遍历中的应用

  • 树的层次遍历；
  • 图的广度优先遍历；
  • 操作系统中先来先服务策略。

特殊矩阵的压缩存储

对称矩阵的压缩存储

策略：只存储主对角线 + 下（上）三角区

按行优先原则将各元素存入一维数组中。

数组的大小为：$\frac{(1+n)*n}{2}$

数组下标和行号列号之间的映射：

​ key：按行优先的原则，$a_{ij}$是第几个元素？

​ $k=\frac{i(i-1)}{2}+j-1,i\geq j$

​ $k=\frac{j(j-1)}{2}+i-1,i<j$

稀疏矩阵的压缩存储

策略：

  • 顺序存储——三元组<行，列，值>
  • 链式存储——十字链表法

其他

数据结构的三要素：逻辑结构、数据的运算、存储结构（物理结构）。存储结构不同，运算的实现方式不同。
